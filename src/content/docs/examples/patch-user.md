Метод `PATCH /users/{id}` позволяет обновить отдельные поля пользователя, не передавая объект целиком. В теле запроса достаточно указать только те поля, которые нужно изменить — остальные останутся прежними.

## Запрос

Допустим, у пользователя с `id = 42` нужно поменять адрес и перевести аккаунт в статус `suspended`. Передавать остальные поля не нужно.

```bash
curl -X PATCH https://api.documentat.io/api/prod/users/42 \
  -H "Token: $API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "accountStatus": "suspended",
    "address": {
      "country": "Россия",
      "city": "Санкт-Петербург",
      "street": "Невский проспект",
      "house": "10"
    }
  }'
```

## Успешный ответ

В случае успеха сервер вернёт `200 OK` и полный объект пользователя с обновлёнными полями:

```json
{
  "id": 42,
  "username": "ivan",
  "email": "ivan@example.com",
  "firstName": "Иван",
  "lastName": "Иванов",
  "age": 30,
  "isEmployee": true,
  "accountStatus": "suspended",
  "address": {
    "country": "Россия",
    "city": "Санкт-Петербург",
    "street": "Невский проспект",
    "house": "10"
  }
}
```

## Возможные ошибки

| Код | Когда возникает                                                |
| --- | -------------------------------------------------------------- |
| 401 | Заголовок `Token` отсутствует или токен невалиден.             |
| 404 | Пользователь с указанным `id` не найден.                       |
| 405 | Метод не разрешён для этого ресурса.                           |
| 422 | Тело запроса не прошло валидацию (например, неизвестное поле). |
| 500 | Внутренняя ошибка сервера — повторите запрос позднее.          |

## Отличие от PUT

`PATCH` обновляет только переданные поля, тогда как `PUT /users/{id}` ожидает полный объект пользователя и заменяет запись целиком. Если нужно изменить одно-два поля — используйте `PATCH`.
