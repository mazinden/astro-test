## Aside

`Aside` бывает четырёх типов:

<Aside type="note">
  По умолчанию — нейтральная заметка. Подходит для дополнительного контекста.
</Aside>

<Aside type="tip" title="Совет">
  Сохраняйте токен в переменной окружения, а не в коде.
</Aside>

<Aside type="caution">
  При смене `email` через `PATCH` пользователю придётся подтвердить новый адрес.
</Aside>

<Aside type="danger">
  Никогда не публикуйте `Token` в публичных репозиториях — отзывайте скомпрометированные токены немедленно.
</Aside>

## Steps

<Steps>

1. Получите персональный токен и сохраните его в переменную окружения.

2. Сделайте `POST /users` с минимальным набором обязательных полей.

3. Сохраните `id` из ответа — он понадобится для последующих запросов.

4. Проверьте создание через `GET /users/{id}`.

</Steps>

## Badge

Метки полезны рядом с пунктами меню или заголовками. Поддерживаются варианты `note`, `tip`, `danger`, `caution`, `success`.

- Эндпоинт <Badge text="Новое" variant="success" /> добавлен в версии 1.2.
- Метод <Badge text="Beta" variant="caution" /> может измениться без предупреждения.
- Поле <Badge text="Deprecated" variant="danger" /> будет удалено в версии 2.0.
- Раздел <Badge text="Draft" variant="note" /> ещё не финализирован.

Размер задаётся через `size`:

<Badge text="small" size="small" />
<Badge text="medium" size="medium" />
<Badge text="large" size="large" />
